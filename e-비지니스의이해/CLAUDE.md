# e-비즈니스의 이해 - 프로젝트 가이드

## 역할
- 대학생활 공부 파트너
- 과목: e-비즈니스의 이해

## 워크플로우: 주차별 학습자료 제작

### 입력
- `N주차 - 제목.txt` 파일에 수업 노트가 작성됨
- 사용자가 "N주차 내용 업데이트했어"라고 하면 txt 파일을 다시 읽고 HTML에 반영

### 출력: `N주차_학습자료.html`
아래 구성요소를 포함하는 인터랙티브 학습 페이지를 생성:

1. **핵심 요약 (30초 정리)** - 키워드 정의 박스로 핵심 개념을 한눈에 정리
2. **개념 비교 카드** - 대비되는 개념이 있을 경우 나란히 비교 (예: e-Commerce vs e-Business)
3. **관계 다이어그램** - 개념 간 포함/관계를 시각적으로 표현
4. **타임라인** - 역사/발전 과정이 있을 경우 연대기 형태로 정리
5. **실생활 예시 카드** - 수업 내용을 실제 기업/서비스에 매핑
6. **셀프 체크 퀴즈** - 수업 내용 기반 객관식 문제, 정답/오답 피드백 포함

### 디자인 규칙
- **기본: 라이트 테마** (배경: #f3f4f6, 카드: white)
- **야간 모드 토글 버튼** 네비게이션 우측에 항상 포함 (🌙/☀️ 전환)
  - `data-theme="dark"` 속성으로 전환, localStorage에 설정 저장
  - 다크 배경: #0f172a, 카드: #1e293b
- 상단 헤더: 파란 그라데이션 (linear-gradient(135deg, #1e3a8a, #2563eb))
- CSS 변수로 색상 관리 (--primary, --success, --danger 등)
- 카드 border-top 색상으로 섹션 구분
- 상단 고정 네비게이션 바 (섹션 앵커 링크)
- 모바일 반응형 (grid 2열 → 1열)
- 단일 HTML 파일 (외부 JS/CSS 의존 없음, 폰트 제외)
- 연습문제 정답 확인은 JavaScript로 인터랙티브하게 처리

### 업데이트 규칙
- txt 파일에 새 내용이 추가되면 기존 HTML에 **증분 반영** (Edit 도구 사용)
- 수업 노트 원문을 정확히 반영하되, 이해를 돕는 부연 설명 추가 가능
- 새 내용에 맞는 퀴즈 문항도 함께 추가
- 작업 완료 후 HTML을 자동으로 열지 않음 (사용자가 별도 요청 시에만 열기)

### 파일 구조
```
e-비지니스의이해/
├── CLAUDE.md
├── 1주차 - 전자상거래란 무엇인가.txt    (수업 노트)
├── 1주차_학습자료.html                   (생성된 학습 페이지)
├── 2주차 - ....txt                       (향후 추가)
├── 2주차_학습자료.html                   (향후 생성)
└── ...
```
